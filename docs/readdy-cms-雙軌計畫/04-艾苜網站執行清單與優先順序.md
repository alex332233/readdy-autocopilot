1. 先完成 `P0 先修 Bug`
2. 再做 `P1 CMS 基礎能力`

4. 最後處理 `P3 SEO 與上線準備`

## 備註

- 目前前端有 staging / preview 與 production 的區分，但 Sanity Studio 與 dataset 仍是共用。
- 因此前端程式碼可在 staging 驗證後再 merge `main`，但 Studio schema / structure / document model 的變動仍需保守安排。
- 若某些 CMS 結構調整會直接影響正式內容維護流程，建議標記為接近上線再切換。
- 文章編輯器目前已採「新版 `body` + 舊欄位 fallback」雙軌模式。新文章可直接使用新版編輯器，舊文章不需立即搬移。
