# PM 測試回饋修正清單

本文件整理 PM 測試版回饋的第一輪修正項目，作為後續一項一項實作與驗收的 checklist。

目前原則：

- 先修低風險且畫面明確的特色療程頁問題，再處理全站 taxonomy 架構。
- 每一項改完後單獨驗收，避免 Studio schema、前端 fallback、Sanity draft / published 狀態互相干擾。
- 不覆蓋 Readdy 原始更新資料夾到正式前端。

- 經驗寫回範圍不限 taxonomy，也包含特色療程頁色彩 token、共用卡片資料來源、Presentation `data-sanity` 對應、Studio structure 設計、reference selector UX、migration/audit 流程、日期格式清理、真實見證/衛教文章架構一致化等。

- 若經驗與文章編輯器、slug、SEO、日期、內容 audit 相關，更新 `05-通用文章編輯器升級規格與艾苜落地方案.md`。
- 若本輪產生新的通用模式，可在 `06-PM測試回饋修正清單.md` 或新文件中新增「本輪可自動化經驗」章節。
- 經驗紀錄要寫「為什麼這樣做」、「避免了什麼坑」、「未來工具可以怎麼自動化」，不要只寫改了哪些檔案。
