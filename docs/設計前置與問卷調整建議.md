# 設計前置與問卷調整建議

更新日期：2026-03-14

## 背景
目前設計師在前期花費不少時間做以下工作：
- 閱讀客戶填寫的品牌需求表單
- 到官網、FB、IG、Google 商家、LINE 等來源補資料
- 交叉比對資訊是否一致
- 將分散的資料重新整理成可討論的品牌 brief

真正耗時的往往不是填表本身，而是後續的搜尋、驗證、重整與追問。

## 核心判斷
這份品牌需求表單內容完整，但混合了三種不同目的：
- 品牌基本資料蒐集
- 品牌策略與設計訪談
- 專案交付項目確認

因此它更適合做為「第二階段確認表單」，而不是第一個客戶入口。

更有效率的做法應該是：
- 第一階段讓客戶填一份更短的 intake
- 系統自動抓取並整理公開資料
- 設計師再依整理後的資訊做確認與深化訪談

## 目前表單的主要問題
### 1. 問題類型混在一起
- 基本資料、品牌策略、名片資訊、CIS 項目需求都放在同一份表單
- 客戶填寫時需要頻繁切換思考脈絡，容易疲勞

### 2. 開放題過多
- 很多題目需要長文回答
- 客戶填答品質容易不一致
- 設計師最後仍需自行重整

### 3. 缺少資料來源結構化欄位
- 目前只有「公司 / 品牌官網或社群連結」
- 但實際上設計師會需要分開知道：
- 官網
- Facebook
- Instagram
- Google 商家
- LINE 官方帳號

### 4. 缺少可自動抓取入口
- 表單沒有被設計成能直接支持後續自動化資料彙整
- 因此設計師仍需手動搜尋與補資料

### 5. 部分欄位標示與邏輯不一致
- 例如標示「選填」但實際又是必填
- 這會影響客戶填答體驗，也會增加客服與設計師後續說明成本

## 問卷內容可如何拆分
### A. 可自動抓取或半自動補齊的資訊
這類資訊可以由系統根據客戶提供的連結先做初步整理：
- 公司名稱
- 品牌中文名稱 / 英文名稱
- 官網
- FB / IG / LINE / Google 商家
- 聯絡電話
- Email
- 地址 / 營業時間
- 服務項目
- 社群簡介
- 近期貼文主題
- 參考品牌網址

### B. 必須由客戶親自回答的主觀策略問題
這類資訊不適合直接自動生成為正式答案：
- 品牌名稱由來
- 品牌核心精神
- 願景
- 創立契機與初衷
- 未來五年服務擴展規劃
- 五年後希望品牌帶給顧客的感受
- 不喜歡的品牌與風格元素

### C. 更適合改為設計師確認的欄位
這類問題客戶通常能提供方向，但最終仍需設計師重新整理：
- 與市場相比的定價區間
- 主要目標族群描述
- 市場差異與獨特優勢
- 品牌風格與設計調性
- 喜歡的參考品牌與原因

### D. 應作為專案範圍確認的欄位
這類資訊可以保留，但建議獨立成另一段：
- CIS 設計項目
- 名片 / 簡報 / 社群模板 / 網站需求
- 最後補充說明

## 建議的新流程
### 第一階段：精簡 intake
客戶只需先填以下資料：
- 品牌名稱
- 官網
- FB
- IG
- Google 商家
- LINE
- 主要服務
- 聯絡人與 Email
- 本次需要的交付項目

### 第二階段：系統自動彙整
系統根據公開資料產出初步品牌摘要，包含：
- 基本品牌資料卡
- 服務內容摘要
- 品牌語氣與風格線索
- 目標族群候選描述
- 通路與社群現況
- 近期貼文與溝通方向摘要
- 可用素材與缺漏素材
- 建議首頁區塊與 CTA
- 缺漏資訊清單

### 第三階段：設計師確認與深化
設計師不再從零開始搜資料，而是：
- 檢查系統彙整是否正確
- 補上主觀策略問題
- 和客戶確認品牌方向
- 產出網站 / CIS brief

## 建議保留的正式訪談題目
以下題目仍有價值，建議保留作為後續確認表單：
- 品牌名稱由來
- 核心精神
- 願景
- 創立初衷
- 服務特色與市場差異
- 未來 3-5 年規劃
- 目標族群
- 希望品牌帶給顧客的感受
- 喜歡 / 不喜歡的品牌與風格
- 專案交付範圍

## 建議改由系統先帶入的欄位
以下資料可由系統先整理，再由設計師或客戶確認：
- 官網與社群連結
- 聯絡資訊
- 地址與營業時間
- 品牌簡介
- 服務項目候選
- 社群溝通主題
- 可用視覺素材
- 基本競品與參考品牌整理

## 最有價值的三份輸出
如果要產品化，重點不只是問卷，而是把問卷轉成可工作的成果物。

最值得產出的三份內容：
- 品牌摘要卡
- 網站 brief
- 缺漏資訊 checklist

這三份文件比原始問卷更接近設計師實際工作流程，也更適合作為後續網站生成與內容整理的起點。

## 建議結論
### 短期
- 保留現有表單作為品牌訪談參考
- 先把明顯不一致的欄位修正，例如選填卻被設成必填

### 中期
- 新增一份更短的客戶 intake 表單
- 將目前表單改為第二階段確認用

### 長期
- 導入公開資料抓取與品牌自動彙整
- 讓設計師從資料蒐集者轉為品牌判讀與提案者

## 總結
目前這份表單不是不能用，而是位置放錯了。

它適合成為「確認與深化」工具，不適合作為「第一個客戶入口」。

若把入口做短、資料先自動彙整、再由設計師確認，會比現在更省時間，也更有機會產品化。
